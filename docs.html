Coordinate Transformation (JSON + CSV)

This page explains exactly how bbox coordinates become overlay pixels in the viewer.

Inline PDF Reference

Sample PDF is embedded here so you can compare coordinates while reading formulas.

Pipeline

  • Parse detection source (JSON or CSV).
  • Normalize to bbox object: x1,y1,x2,y2,page.
  • Choose mapping mode (auto/manual).
  • Transform bbox into viewport pixel rectangle.
  • Draw rectangle on overlay canvas.
viewport = pdf.js page.getViewport({ scale })
rect = transform(bbox, viewport, mappingMode)
overlayCtx.strokeRect(rect.x, rect.top, rect.width, rect.height)

Page navigation

  • Prev/Next buttons
  • Page dropdown
  • [ ] or Arrow Left/Right

JSON Transform

Primary mode uses pdf.js convertToViewportPoint (PDF-space to screen-space).

p1 = viewport.convertToViewportPoint(x1, y1)
p2 = viewport.convertToViewportPoint(x2, y2)
left = min(p1.x, p2.x), top = min(p1.y, p2.y)
width = abs(p2.x - p1.x), height = abs(p2.y - p1.y)

CSV Transform

CSV rows use columns x0,y0,x1,y1. The viewer can apply multiple mapping variants.

Mode Meaning When to use
csv-raw-top-left Interpret CSV as top-left page units Best when CSV already matches PDF geometry
csv-raw-top-left-yflip Raw top-left with Y inversion When CSV Y-axis appears inverted
csv-raw-pdf-origin Treat CSV as PDF-origin values When source is already PDF-space
csv-fit-* Normalize CSV extents to viewport When CSV is relative/trimmed or from raster extents

Worked Example

Example row from sample CSV:

page=1, x0=384, y0=285, x1=510, y1=288

With page size 3024x2160 and raw-top-left mapping at scale 1.0:

rect.x = 384
rect.top = 285
rect.width = 126
rect.height = 3
Field Formula Value
x min(x0, x1) min(384,510) = 384
top min(y0, y1) min(285,288) = 285
width abs(x1 - x0) abs(510-384) = 126
height abs(y1 - y0) abs(288-285) = 3

Cursor Coordinates

Viewer always shows both coordinate spaces:

cursor px: (cx, cy) | pdf: viewport.convertToPdfPoint(cx, cy)

Troubleshooting Misaligned Boxes

  • If all CSV boxes are shifted but shape is correct: try csv-fit or csv-fit-uniform.
  • If boxes look mirrored vertically: try csv-raw-top-left-yflip or csv-fit-uniform-yflip.
  • If boxes are tiny/huge: verify CSV units and use csv-raw-top-left for page-unit data.
  • Use status line mapping scores to see which auto mode was selected and why.
